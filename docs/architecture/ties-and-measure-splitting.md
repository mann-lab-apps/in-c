# 타이와 마디 경계 음가 분할

## 모델

단성부 타이는 `Note.ties.start`와 `Note.ties.stop`으로 저장한다. 별도
대상 ID를 저장하지 않고 악보 시간축에서 바로 인접한 이벤트를 관계의
대상으로 사용한다.

유효한 타이는 다음 조건을 모두 만족한다.

- 시작과 종료 이벤트가 모두 음표다.
- 두 이벤트가 단성부 시간축에서 바로 인접한다.
- 두 음표의 기보 음이름, 옥타브와 유효 `alter`가 같다.
- 시작 음표에는 `start`, 종료 음표에는 `stop`이 함께 존재한다.

이 구조는 MusicXML의 `tie`와 `notations/tied` 표현에 직접 대응한다.

## 생성과 해제

`buildTieCommand()`는 선택한 시작 음표와 다음 이벤트를 검증한 뒤 양쪽
플래그를 함께 변경한다. 같은 마디는 하나의 `voice-events.replace`,
마디 경계는 두 교체를 포함한 `score.batch`로 실행한다. 따라서 타이
생성·해제와 undo/redo가 항상 원자적이다.

렌더러에서는 툴바의 타이 버튼 또는 `L` 단축키로 입력·해제한다. 단축키는
물리 키 코드(`KeyL`)를 기준으로 처리하므로 한글 입력 상태에서도 동일하게
동작한다.

- 타이가 없는 음표를 선택하면 바로 다음 이벤트가 같은 음높이의 음표일 때
  타이를 생성한다.
- 타이의 시작 음표나 종료 음표를 선택하면 해당 연결을 해제한다.
- 쉼표, 다른 음높이, 중간에 다른 이벤트가 끼어 있는 경우에는 타이를 만들지
  않는다.
- 슬러는 다른 음 사이도 연결할 수 있는 별도 기호이므로 이 모델에 섞지 않고
  별도 이슈에서 다룬다.

타이로 연결된 음표 삭제 정책은 `#110`에서 따로 정리한다. 현재 삭제 동작은
무효한 타이 관계를 만들지 않도록 보수적으로 막고, 사용자는 먼저 타이를
해제한 뒤 삭제한다.

## 마디 경계 입력

순차 음표 입력의 duration이 현재 마디를 넘으면 다음 순서로 처리한다.

1. 전체 tick 범위를 마디 경계로 나눈다.
2. 각 마디 구간을 표기 가능한 duration 조각으로 분해한다.
3. 첫 입력 음의 실제 음높이를 모든 조각에 복사한다.
4. 조각 사이에 `start`와 `stop`을 설정한다.
5. 악보 끝을 넘으면 앞 마디의 clef, key signature, time signature를
   상속한 빈 마디를 생성한다.
6. 모든 이벤트 교체와 마디 생성을 하나의 `score.batch`로 묶는다.

쉼표는 타이로 연결할 수 없으므로 마디 경계를 넘는 쉼표 입력은 현재
거부한다.

## 출력

- VexFlow은 같은 시스템의 타이를 `StaveTie` 하나로 그린다.
- 시스템 경계를 넘는 타이는 앞 시스템의 열린 타이와 다음 시스템의
  닫힌 타이로 나누어 그린다.
- 재생 시간축은 연결된 음표를 하나의 지속 이벤트로 병합한다.
- MusicXML은 재생용 `tie`와 기보용 `notations/tied`를 함께
  import/export한다.
