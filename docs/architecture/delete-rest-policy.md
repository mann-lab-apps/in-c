# 삭제와 쉼표 병합 정책

## 목적

`in C`의 단성부 편집 모델은 각 마디가 항상 정확히 채워져 있어야 한다. 따라서
음표를 지운다고 해서 시간 슬롯이 사라지지는 않는다. 대신 선택한 음표를 같은
구간의 쉼표로 비우고, 인접한 쉼표가 있으면 하나의 긴 쉼표 구간으로 병합한다.

## 기본 규칙

- 음표에서 `Backspace` 또는 `Delete`를 누르면 해당 음표를 쉼표로 비운다.
- 비운 쉼표 앞뒤에 연속된 쉼표가 있으면 가능한 가장 긴 쉼표로 병합한다.
- 병합 구간이 마디 전체를 채우면 full-measure rest로 정리한다.
- 쉼표에서 `Backspace` 또는 `Delete`를 누르면 인접 쉼표가 있을 때 병합한다.
- 병합할 인접 쉼표가 없는 쉼표는 이미 비어 있는 시간 슬롯이므로 변경하지 않고
  상태 메시지를 표시한다.
- 타이로 연결된 음표는 먼저 타이를 해제해야 비울 수 있다.
- tuplet 관계가 깨지는 삭제 또는 병합은 command를 만들지 않는다.

## 선택 이동

음표를 비운 뒤에는 다음 이벤트를 우선 선택한다. 다음 이벤트가 병합으로 사라졌거나
없으면 이전 이벤트를 선택하고, 둘 다 없으면 병합된 쉼표 또는 해당 마디를 선택한다.

이 규칙은 여러 음표를 연속으로 비울 때 같은 키를 반복해서 눌러도 편집 대상이
자연스럽게 앞으로 이동하도록 하기 위한 것이다.

## 사용자에게 보이는 의미

툴바와 키보드 동작은 `Delete event`가 아니라 `Clear note to rest` 또는
`Merge adjacent rests`로 이해되어야 한다. 실제 시간 슬롯을 제거하는 기능은 아직
제공하지 않는다.
