# 제품 결정 로그 형식

작성일: 2026-07-16

## 목적

시장조사, 인터뷰, 사이트 analytics, GitHub 이슈에서 나온 근거를 제품 결정으로 옮길
때 같은 형식으로 남긴다. 결정 로그는 확정 문서가 아니라 당시 근거와 대안을 다시
검토할 수 있게 하는 기록이다.

## 저장 원칙

- 개인정보, 인터뷰 원문, 연락처, 녹취는 포함하지 않는다.
- 수치는 출처, 조회일, 표본/추정/확정 상태를 함께 적는다.
- 결정이 바뀌면 이전 결정을 지우지 않고 새 로그로 갱신한다.

## 템플릿

```md
## YYYY-MM-DD 결정 제목

### 결정

[무엇을 하기로 했는지 한 문장]

### 배경

- 관련 이슈:
- 관련 문서:
- 관찰한 신호:

### 근거

| 근거 | 출처 | 상태 | 조회일 |
| --- | --- | --- | --- |
| [확인 필요] | [문서/이슈/인터뷰 익명 코드] | 확정/표본/추정 | YYYY-MM-DD |

### 대안

| 대안 | 장점 | 보류 이유 |
| --- | --- | --- |
| [확인 필요] | [확인 필요] | [확인 필요] |

### 리스크

- [확인 필요]

### 후속 작업

- [ ] GitHub issue:
- [ ] 문서/사이트 반영:
- [ ] 재검토 날짜:
```

## 결정 분류

| 분류 | 예시 | 재검토 시점 |
| --- | --- | --- |
| 제품 범위 | 베타 모집 위치, Classes 신청 없음 상태 | 베타 30명 또는 관련 이슈 완료 후 |
| 콘텐츠 전략 | Columns 주제 우선순위, 악보별 안내 | 월간 콘텐츠 리뷰 |
| 사업 가설 | Concert Preview 파일럿, Chromatics 유입 지표 | 인터뷰 5건 또는 파일럿 2건 후 |
| 운영 정책 | 개인정보, 백업, issue template | 기능 출시 전 |
