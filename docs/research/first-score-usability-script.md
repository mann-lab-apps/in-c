# 첫 악보 작성 사용성 인터뷰 스크립트

작성일: 2026-07-16

## 목적

Chromatics를 처음 여는 사용자가 새 악보를 만들고, 몇 개의 음표와 쉼표를 입력한 뒤,
MusicXML로 저장하는 흐름을 30분 안에 관찰한다. 목표는 사용자를 평가하는 것이 아니라
UI 문구, 빈 상태, 실패 메시지, 저장 흐름에서 어디가 막히는지 찾는 것이다.

## 개인정보 원칙

- 실명, 연락처, 녹취, 원본 화면 녹화는 레포에 저장하지 않는다.
- 레포에는 익명 코드, 대상군, 막힌 단계, 요약 인사이트만 남긴다.
- 사용자가 직접 만든 비공개 악보 파일은 GitHub issue나 레포에 첨부하지 않는다.

## 진행 구성

| 구간 | 시간 | 진행 |
| --- | --- | --- |
| 안내와 동의 | 3분 | 목적, 익명 기록, 중단 가능성을 안내한다 |
| 첫 인상 | 3분 | 시작 화면에서 무엇을 누를지 말하게 한다 |
| 과제 수행 | 15분 | 새 악보, 음표/쉼표 입력, 저장까지 직접 해본다 |
| 회고 질문 | 7분 | 막힌 지점과 다음에 필요한 도움을 묻는다 |
| 정리 | 2분 | 공개 제보 시 개인정보 제외 원칙을 다시 안내한다 |

## 과제

1. 새 악보를 만든다.
2. 제목, 작곡가, 박자표, 조표, 빠르기를 확인하거나 바꾼다.
3. 4분음표 4개 또는 익숙한 짧은 멜로디를 입력한다.
4. 쉼표 하나를 음표로 바꾸거나 음표 하나를 쉼표로 바꾼다.
5. 입력한 내용을 재생해본다.
6. MusicXML로 저장하고 다시 열 수 있는지 확인한다.

## 관찰 항목

| 단계 | 볼 것 | 기록 방식 |
| --- | --- | --- |
| 시작 화면 | 새 악보, 예제, 최근 파일의 의미를 구분하는가 | 막힘 없음/질문함/오해함 |
| 악보 설정 | 제목, 박자표, 조표, 빠르기 문구를 이해하는가 | 사용자가 읽은 문구 |
| 입력 | 음가 선택, 음표 입력, 쉼표 입력 순서를 찾는가 | 첫 막힘 단계 |
| 수정 | 실수 후 되돌리거나 선택을 바꾸는가 | 복구 행동 |
| 저장 | MusicXML 저장과 다시 열기 의미를 이해하는가 | 성공/실패/확인 필요 |
