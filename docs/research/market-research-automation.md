# Market Research Automation Design

작성일: 2026-07-12

## 목적

시장조사 자동화 후보를 구현 전에 설계한다. 이 문서는 무엇을 자동화할지보다 무엇을
자동화하지 않을지를 먼저 정한다.

## 자동화 원칙

- 자동화는 수동 루틴이 2-3회 반복된 뒤 병목이 확인된 항목에만 적용한다.
- 공개 API, 공식 다운로드, 직접 관리 데이터, GitHub issue 자동화가 우선이다.
- 약관 확인 전 외부 사이트 scraping을 구현하지 않는다.
- 수집 결과는 사람이 검토한 뒤 문서에 반영한다.
- `확정`이 아닌 수치를 사업계획서 문장으로 자동 승격하지 않는다.

## 공연 일정과 공연 수 수집 설계 (#293)

| 항목 | 설계 |
| --- | --- |
| 1차 입력 | KOPIS 공개 API/다운로드 가능 여부 [확인 필요] |
| 2차 입력 | 주요 공연장 일정 페이지와 예매처 공개 상세페이지 |
| 저장 후보 | `docs/research/data/concert-samples.csv` 또는 `classical-market-research.md` 표 |
| 최소 필드 | 공연명, 날짜, 공연장, 지역, 장르, 규모, 출처 URL, 조회일, 확인 상태 |
| 실행 주기 | 월간 표본 기록, 분기 공식 통계 확인 |
| 선행 조건 | 출처 카탈로그 등록, 약관/API/robots 확인, 개인정보 미수집 확인 |
| 반영 위치 | #281 국내 공연 규모, #282 공연장 규모별 구조, #295 홍보 채널 표본 |

### 수집 후보 비교

| 방법 | 장점 | 제한 | 초기 판단 |
| --- | --- | --- | --- |
| KOPIS API/다운로드 | 공식성과 반복성 높음 | 접근 조건과 장르 분류 확인 필요 | 최우선 확인 |
| 공연장 일정 페이지 수동 표본 | 맥락과 홍보 자산을 함께 볼 수 있음 | 표본 편향, 반복 수작업 | 월간 10건 표본으로 시작 |
| 예매처 공개 목록 수동 표본 | 가격/상세페이지 확인 가능 | 약관 확인 전 자동화 금지 | 링크 기록 중심 |

### 실패 시 수동 대체

API 접근이나 자동화 조건이 불명확하면 그 달에는 표본 공연 10건만 수동 기록한다.
수동 표본도 공식 통계처럼 쓰지 않고 `표본`으로 표시한다. 자동화 구현은 약관과 API 조건이
확인된 뒤 별도 이슈로 분리한다.

## 자동화 전 승인 체크

- [ ] 출처 카탈로그에 등록된 자료원인가?
- [ ] 약관/API/robots 확인 상태가 기록되어 있는가?
- [ ] 수집 데이터에 개인정보가 없는가?
- [ ] 원문 복제가 아니라 메타데이터와 링크 기록으로 충분한가?
- [ ] 실패했을 때 수동 대체 방법이 있는가?
- [ ] 사람이 검토한 뒤 문서에 반영하는 흐름인가?

## 데이터 파일 후보

아직 구현하지 않는다. 필요가 반복 확인되면 다음 후보를 검토한다.

- `docs/research/data/venues.csv`
- `docs/research/data/concert-samples.csv`
- `docs/research/data/keyword-trends.csv`
- `docs/research/data/source-catalog.csv`

데이터 파일을 추가할 때도 개인정보, 인터뷰 원문, 저작권 자료 원문은 저장하지 않는다.
