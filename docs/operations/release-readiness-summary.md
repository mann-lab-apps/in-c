# 비개발자용 출시 준비 요약

기준일: 2026-07-15  
근거 범위: 당시 `main`의 Electron 앱, 정적 사이트, 품질 문서

## 한눈에 보는 결론

현재 판정은 **조건부 가능**이다. 앱과 사이트의 핵심 자동 검증은 통과했으므로 내부
테스트용 사전 출시 후보로 제시할 수 있다. 다만 아래 제한을 사용자에게 알리고, 실제
출시 후보에서 사람의 수동 확인을 마친 뒤 담당자가 승인해야 한다.

| 판정 | 범위 | 운영 판단 |
| --- | --- | --- |
| 출시 가능 | 단일 파트·단성부 중심 악보 편집, MusicXML 일부 가져오기·내보내기, 정적 사이트 | 최신 출시 후보에서 자동 검증과 수동 확인을 다시 통과하면 공개할 수 있다. |
| 조건부 가능 | 앱과 사이트 전체 | 알려진 제한을 릴리즈 노트에 적고 담당자 승인 후 진행한다. |
| 보류 | 운영 백엔드, 완성형 다중성부·다중파트 편집, Windows 개발 환경 검증 | 연결 이슈가 해결되거나 별도 검증이 끝날 때까지 지원 범위에 넣지 않는다. |

이 문서는 2026-07-15의 품질 근거를 쉽게 풀어쓴 요약이다. 최신 출시 결정에는 반드시
최신 브랜치에서 [출시 준비 체크리스트](../quality/release-readiness-checklist.md)를 다시
확인한다.

## 지금 보여줄 수 있는 것

- 단일 파트와 단성부를 중심으로 음표·쉼표와 주요 악상 기호를 편집하는 흐름
- 지원 범위 안의 MusicXML 가져오기·내보내기와 악보 렌더링
- 작품·공연·창작자·수업 콘텐츠를 제공하는 정적 사이트

자동 검증 결과는 통과했지만, 이것만으로 실제 설치 파일과 운영 사이트까지 확인된 것은
아니다. 기능별 근거는 [검증 연결표](../quality/traceability-matrix.md)에서 확인한다.

## 제한하거나 숨겨야 하는 것

| 범위 | 사용자에게 알릴 말 | 연결 이슈 |
| --- | --- | --- |
| 계정·커뮤니티·서버 데이터 기능 | 운영 백엔드가 아직 연결되지 않아 현재 지원하지 않는다. | [#316](https://github.com/mann-lab-apps/in-c/issues/316) |
| 같은 오선의 여러 독립 성부 | 일부 내부 모델만 있고 완성된 편집 흐름은 지원하지 않는다. | [#93](https://github.com/mann-lab-apps/in-c/issues/93) |
| 여러 악기 파트와 합주보 | 단일 파트가 현재 지원 범위이며 완성형 합주보 편집은 지원하지 않는다. | [#94](https://github.com/mann-lab-apps/in-c/issues/94) |
| Windows 개발 환경 | 낮은 등급의 개발 서버 경고 영향을 Windows에서 아직 확인하지 않았다. | [#8](https://github.com/mann-lab-apps/in-c/issues/8) |

제품 상태 표시와 자동 검증 범위의 문서 정렬은 각각
[#321](https://github.com/mann-lab-apps/in-c/issues/321),
[#322](https://github.com/mann-lab-apps/in-c/issues/322)에서 추적한다. 이 두 항목은 현재
지원 기능을 새로 추가하는 작업이 아니라, 설명과 검증 범위를 맞추는 후속 작업이다.

## 출시 전에 사람이 확인할 것

1. 패키징된 앱을 설치하고 열어 작은 악보 하나를 만든 뒤 저장·다시 열기·PDF 저장을 확인한다.
2. 실제 운영 주소에서 주요 페이지와 다운로드 링크를 확인한다.
3. Chrome·Safari·Firefox에서 주요 화면과 링크를 확인한다.
4. 릴리즈 노트에 위 제한과 이번 변경을 과장 없이 적었는지 확인한다.

하나라도 핵심 저장·열기 실패, 앱 종료, 보안 문제, 운영 주소 실패가 확인되면 출시를
보류한다. 수동 확인을 실행하지 않은 항목은 통과로 간주하지 않는다.
