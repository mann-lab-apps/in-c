# AI Agent 협업 워크플로우

이 문서는 작업자와 AI Agent가 `docs/product/story-map.md`를 기준으로 기능 개발,
문구 수정, 문서 갱신, 검증 보강을 함께 진행하기 위한 절차다.

## 원칙

- 스토리 맵은 현재 합의된 사용자 여정과 제품 기준이다.
- 이슈와 PR은 변경 배경, 논의, 히스토리를 남기는 곳이다.
- 구현은 스토리 기준을 만족시키는 방향으로 진행한다.
- 스토리 기준이 바뀌면 관련 이슈에 이유를 남기고 `story-map.md`를 함께 수정한다.
- 검증은 스토리의 기대 결과를 보호하는지 기준으로 선택한다.

## 작업 시작 전

1. 현재 브랜치와 워킹트리를 확인한다.
2. 관련 GitHub 이슈의 목표와 완료 조건을 읽는다.
3. `docs/product/story-map.md`에서 영향받는 스토리 ID를 찾는다.
4. 관련 스토리의 사용자, 목표, 흐름, 기대 결과, 지원 상태, 검증 연결을 확인한다.
5. 작업이 기존 스토리 안에 있는지, 새 스토리가 필요한지 판단한다.
6. 사용자 흐름을 바꾸는 작업이면 이슈 본문이나 댓글에 변경 의도를 남긴다.

작업 시작 전 판단은 다음 중 하나로 정리한다.

- `스토리 영향 없음`: 내부 리팩터링이나 사용자 흐름이 바뀌지 않는 변경.
- `영향 스토리: STORY-MVP-002`: 기존 스토리 기준 안에서 진행하는 변경.
- `새 스토리 필요`: 현재 스토리 맵으로 설명하기 어려운 새 사용자 흐름.

## 작업 중

1. UI 문구, 상태 메시지, 오류 메시지가 사용자의 다음 행동을 알려주는지 확인한다.
2. 선택 상태, 입력 상태, 재생 상태처럼 사용자가 직접 체감하는 상태 전이가
   스토리 흐름과 맞는지 확인한다.
3. 구현 중 제품 기준이 바뀌면 코드를 먼저 확정하지 말고 관련 이슈 댓글에
   변경 이유를 남긴다.
4. 기준 변경이 확정되면 `docs/product/story-map.md`의 현재 기준과 지원 상태를 함께 갱신한다.
5. 기존 검증 스크립트가 변경된 흐름을 보호하는지 확인한다.
6. 보호되지 않는 핵심 흐름이면 단위 테스트, MVP 검증, 패키징 검증, 수동 확인
   중 적절한 검증을 추가하거나 PR에 남긴다.

## PR 전

1. `git diff`로 사용자 흐름, 문서, 테스트 변경을 함께 확인한다.
2. `story-map.md`의 현재 기준과 실제 동작이 어긋나지 않는지 확인한다.
3. 스토리 기준을 바꿨다면 관련 이슈에 이유가 남아 있는지 확인한다.
4. PR 본문에 영향받은 스토리 ID를 적는다.
5. 스토리 맵 변경 여부를 적는다.
6. 검증 결과를 적는다.

PR의 `스토리 영향도` 섹션은 다음 중 하나로 작성한다.

```md
## 스토리 영향도

- 영향 스토리: STORY-MVP-003
- 스토리 맵 변경: 있음
- 변경 이유: #123에서 삭제 정책 기준을 변경하기로 결정했습니다.
```

또는:

```md
## 스토리 영향도

- 영향 스토리: 스토리 영향 없음
- 스토리 맵 변경: 없음
- 변경 이유: 내부 리팩터링입니다.
```

## 머지 후

1. 닫힌 이슈가 상위 트래킹 이슈에 포함되어 있으면 체크박스를 갱신한다.
2. 후속 이슈가 필요하면 기존 이슈 댓글 또는 새 이슈로 남긴다.
3. 로컬 `main`을 원격과 맞춘다.
4. 다음 작업 후보를 고를 때 `story-map.md`의 부족함 항목과 열린 이슈를 함께 확인한다.

## 검증 선택 기준

- 순수 문서나 템플릿 변경: `git diff --check`
- score-core, timeline, MusicXML, playback 의미 변경: `npm test`
- renderer layout, 선택 좌표, system 배치 변경: `npm run verify:mvp`
- 패키징, 배포 파일, 설치 앱 흐름 변경: `npm run verify:package`
- 앱 전체 빌드나 Electron/Vite 설정 변경: `npm run build`

여러 영역을 건드리면 해당 검증을 조합한다. 검증을 생략한 경우 PR에 이유를 적는다.

## 자동화 확장 후보

- PR 템플릿의 `영향 스토리` 값이 비어 있으면 경고한다.
- `story-map.md`가 바뀐 PR은 관련 이슈 링크를 요구한다.
- `src/score-core`, `src/io`, `src/playback` 변경 시 `npm test` 실행을 권장한다.
- `src/renderer`, `src/engraving` 변경 시 `npm run verify:mvp` 실행을 권장한다.
- `package.json`, `electron.vite.config.ts`, `.github/workflows` 변경 시 빌드 또는
  CI 영향 확인을 요구한다.
