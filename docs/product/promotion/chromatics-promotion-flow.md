# Chromatics 웹/앱에서 in-C 웹 기능 홍보 흐름

## 목표

Chromatics 사용자가 편집 흐름을 잃지 않으면서 in-C의 웹 편집기, 커뮤니티 악보,
연주회 홍보 기능을 발견하게 한다.

## 노출 위치

| 위치 | 추천 CTA | 우선순위 |
| --- | --- | --- |
| 시작 화면 | 새 웹 기능 보기, 인기 악보 열기 | 높음 |
| 저장 완료 후 | 커뮤니티에 게시하기 | 높음 |
| 시스템 탭 | 계정 연결, API 상태 | 중간 |
| 다운로드 페이지 | Chromatics로 열기 | 높음 |
| 편집기 사이드 영역 | 관련 Columns 또는 작품 보기 | 낮음 |

편집 중 자동 팝업은 사용하지 않는다.

## CTA 설계

- 웹 편집기: `브라우저에서 이어서 편집`
- 커뮤니티 악보: `공개 악보 둘러보기`
- 연주회 홍보: `공연 홍보 등록`
- Columns: `이 작품을 듣는 힌트 보기`

CTA는 외부 브라우저로 열고, 앱에는 현재 문서 저장 상태를 먼저 확인하게 한다.

## 노출 빈도

- 시작 화면 배너는 세션당 1회 노출한다.
- 사용자가 닫은 배너는 같은 `banner.id`에 대해 7일 동안 숨긴다.
- 저장 완료 후 CTA는 파일 저장 성공 직후에만 보여준다.
- 결제 또는 로그인 CTA는 편집 중 자동으로 노출하지 않는다.

## 문구 톤

Chromatics는 `in-C의 무료 악보 편집 도구`로 설명한다. 웹 기능을 홍보할 때도
앱이 부족하다는 뉘앙스보다 `웹에서 공유와 발견을 이어간다`는 흐름을 사용한다.

## 후속 구현

- 시작 화면 배너 슬롯
- 저장 완료 후 커뮤니티 게시 CTA
- 닫기/다시 보지 않기 로컬 상태
- `banner_impression`, `banner_click`, `banner_dismiss` 연결
