# Electron 앱과 웹 공통 홍보 배너 슬롯

## 슬롯 위치

| Surface | Placement | 용도 |
| --- | --- | --- |
| Electron 시작 화면 | `start_screen` | 릴리즈 공지, 웹 기능 안내, 인기 악보 |
| Electron 시스템 탭 | `system_tab` | 계정/API 상태, 공지 |
| 웹 랜딩 | `home` | 공연 프리뷰, Columns 연결 |
| 다운로드 페이지 | `download` | Chromatics 역할 안내 |
| 편집기 사이드 | `editor_side` | 관련 작품/Columns, 저장 후 CTA |

## 배너 종류

- 연주회 홍보
- 새 웹 기능 안내
- 커뮤니티 악보 추천
- 릴리즈 공지
- 점검 또는 API 상태 공지

## UX 규칙

- 배너는 한 줄 CTA와 닫기 버튼을 가진다.
- `다시 보지 않기`는 같은 `banner.id`에만 적용한다.
- 외부 링크는 새 브라우저 창에서 연다.
- 유료 홍보는 `프로모션` 또는 동등한 표시를 포함한다.
- 편집기 핵심 버튼보다 배너 CTA를 더 강조하지 않는다.

## 실패와 오프라인

API가 실패하면 슬롯 자체를 숨긴다. 마지막 성공 캐시가 있더라도 만료 시간이
지났으면 표시하지 않는다. 네트워크 오류를 사용자에게 큰 오류로 보여주지 않고,
시스템 탭의 연결 상태에서만 확인할 수 있게 한다.

## 접근성

- 닫기 버튼은 `배너 닫기` aria-label을 사용한다.
- CTA는 목적지를 알 수 있는 문구를 쓴다.
- 자동으로 포커스를 가져가지 않는다.
- 움직이는 배너나 자동 carousel은 사용하지 않는다.
