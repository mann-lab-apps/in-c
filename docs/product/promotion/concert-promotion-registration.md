# 연주회 홍보 등록 UX와 데이터 모델

## MVP 흐름

연주회 홍보 등록은 긴 단일 폼이 아니라 세 단계로 나눈다.

1. 기본 정보: 제목, 날짜와 시간, 장소, 주최자, 연락처를 입력한다.
2. 프로그램: 출연자, 작품, 짧은 소개, 예매 링크, 대표 이미지를 입력한다.
3. 노출 방식: 무료/유료 홍보, 노출 기간, 미리보기, 검수 요청을 확인한다.

사용자는 각 단계에서 임시 저장할 수 있고, 검수 요청 전에는 공개되지 않는다.

## 필수 필드

| 필드 | 설명 |
| --- | --- |
| `title` | 공연명 |
| `startsAt` | 공연 시작 일시 |
| `venueName` | 장소명 |
| `venueAddress` | 주소 또는 지역 |
| `organizerName` | 주최/주관 이름 |
| `contactEmail` | 운영자 연락용 이메일 |
| `ticketUrl` | 예매 또는 상세 정보 링크 |
| `programNote` | 관객에게 보여줄 짧은 소개 |
| `promotionType` | `free` 또는 `paid` |
| `reviewStatus` | `draft`, `submitted`, `approved`, `rejected` |

## 선택 필드

- 출연자와 역할
- 프로그램 작품 목록과 작곡가
- 가격 정보
- 대표 이미지와 대체 텍스트
- 관련 Creator, Columns, 작품 페이지 ID
- 온라인/오프라인 여부

## 무료와 유료 홍보

무료 홍보는 기본 목록 노출과 검색 가능 상태만 제공한다. 유료 홍보는 결제와
검수 통과 후 지정 placement에 추가 노출된다. 유료 홍보도 광고 표시와 노출
기간을 명확히 보여야 한다.

## 상태 모델

| 상태 | 의미 |
| --- | --- |
| `draft` | 작성 중이며 외부에 노출되지 않음 |
| `submitted` | 검수 대기 |
| `changesRequested` | 수정 요청 |
| `approved` | 공개 가능 |
| `rejected` | 공개 불가 |
| `expired` | 공연 종료 또는 노출 기간 종료 |

## 모바일 구성

모바일에서는 입력 필드를 접이식 단계로 나누고, 하단 고정 영역에는 `임시 저장`
과 `다음`만 둔다. 이미지 업로드와 프로그램 입력은 선택 입력으로 두어 첫 등록
완료까지의 부담을 낮춘다.
