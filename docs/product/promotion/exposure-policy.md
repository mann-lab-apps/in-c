# 홍보 콘텐츠 노출 정책과 정렬 기준

## 원칙

in-C의 공연 홍보는 사용자 신뢰를 우선한다. 유료 홍보는 숨기지 않고 명확히
표시하며, 편집자 추천이나 날짜순 콘텐츠와 섞일 때도 노출 이유를 설명한다.

## 콘텐츠 유형

| 유형 | 표시 방식 | 기본 위치 |
| --- | --- | --- |
| 무료 공연 정보 | 일반 카드 | 공연 목록, 검색 결과 |
| 편집자 추천 | 추천 배지 | 홈, Columns 연결 영역 |
| 유료 홍보 | 프로모션 표시 | 지정 배너, 상단 슬롯 |
| 커뮤니티 인기 콘텐츠 | 집계 기준 표시 | 탐색 섹션 |

## 정렬 기준

공연 목록의 기본 정렬은 `다가오는 날짜순`이다. 사용자가 선택할 수 있는 보조
정렬은 다음으로 제한한다.

- 지역
- 장르 또는 편성
- 종료 임박
- 최근 등록
- 편집자 추천

유료 홍보는 기본 정렬 자체를 바꾸지 않고, 별도 슬롯에서 노출한다.

## 만료 정책

- 종료된 공연은 일반 목록에서 숨긴다.
- 관련 Columns나 작품 페이지에서는 `지난 공연`으로 보관할 수 있다.
- 결제 노출 기간이 끝난 유료 홍보는 자동으로 일반 공연 정보로 돌아간다.
- 공연 시간이 불명확하면 검수 단계에서 공개하지 않는다.

## 홈과 탐색 노출

홈 첫 화면에는 제품 메시지, Columns, 작품 중심 탐색을 우선한다. 유료 홍보는
첫 화면 전체를 점유하지 않고, 공연 맥락이 있는 영역에만 둔다. 탐색 페이지는
무료 정보와 유료 홍보를 시각적으로 구분한다.

## 이벤트 계측

노출 정책 검증을 위해 다음 이벤트를 남긴다.

- `promotion_impression`
- `promotion_click`
- `promotion_dismiss`
- `concert_filter_change`

payload에는 placement, contentId, promotionType, sortKey만 포함하고 개인식별
정보는 넣지 않는다.

## 후속 구현

- 노출 API의 `placement`와 `promotionType` 필드 연결
- 홈/탐색/공연 목록 UI 슬롯 구현
- 유료 표시 문구와 접근성 라벨 적용
