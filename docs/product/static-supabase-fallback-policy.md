# 정적 사이트와 Supabase fallback 기준

이 문서는 Supabase 읽기 기능을 붙이기 전에 합의할 사용자 경험과 공개 범위 기준이다.
현재 사이트 동작이나 Supabase 설정은 바꾸지 않는다.

## 인수 기준

- Supabase 요청이 실패하거나 제한 시간 안에 끝나지 않으면 공개 정적 데이터를 보여준다.
- Supabase가 정상적으로 빈 결과를 반환하면 정적 데이터로 바꾸지 않고 빈 상태를 보여준다.
- 정적 데이터로 바뀌어도 사용자는 페이지를 읽고 기존 링크를 계속 사용할 수 있다.
- `draft`와 `private` 콘텐츠는 Supabase 응답과 정적 데이터 모두에서 공개하지 않는다.
- 오래된 정적 데이터를 보여줄 때는 마지막 갱신 기준과 현재 상태를 사용자가 알 수 있다.

## 실패 가능한 검증 기준

- 네트워크 오류나 제한 시간 초과 뒤 페이지 전체가 오류 화면이 되면 실패한다.
- 정상 빈 결과를 네트워크 실패로 오해해 정적 콘텐츠를 보여주면 실패한다.
- `draft` 또는 `private` 콘텐츠가 공개 응답이나 정적 빌드에 포함되면 실패한다.
- fallback 데이터의 기준 시각이 없거나 오래된 상태를 최신 정보처럼 표시하면 실패한다.
- fallback 뒤 페이지의 기존 링크를 사용할 수 없으면 실패한다.
- Markdown 상대 링크 검사 또는 `git diff --check`가 통과하지 않으면 실패한다.

## 상태별 동작

| Supabase 읽기 결과 | 화면에 사용할 데이터 | 사용자 안내 |
| --- | --- | --- |
| 성공, 공개 결과 있음 | Supabase의 공개 결과 | 별도 안내 없음 |
| 성공, 공개 결과 없음 | 빈 목록 | 기존 빈 상태 문구 |
| 네트워크 오류, 서버 오류, 제한 시간 초과 | 저장소의 공개 정적 데이터 | 정적 데이터로 보여 주고 있음을 알림 |
| 응답 형식이 계약과 다름 | 저장소의 공개 정적 데이터 | 정적 데이터로 보여 주고 있음을 알림 |

정상 빈 결과는 장애가 아니다. 빈 결과에서 정적 데이터로 바꾸면 이미 내린 콘텐츠가
다시 나타날 수 있으므로 fallback을 실행하지 않는다. 제한 시간은 read adapter 구현
이슈에서 정하되, 시간 초과는 네트워크 오류와 같은 결과로 처리한다.

## 공개 범위

- 공개 클라이언트는 `status = 'public'`인 항목만 요청하고 표시한다.
- `draft`, `private`, 알 수 없는 상태는 공개 결과에서 제외한다.
- 정적 fallback 파일에도 공개 항목만 넣는다. 비공개 항목을 받은 뒤 화면에서 숨기는
  방식은 사용하지 않는다.
- Supabase 응답과 정적 파일이 모두 사용 불가능하면 빈 상태와 다시 시도 안내를
  표시한다. 오류 세부 정보나 내부 데이터는 화면에 노출하지 않는다.

## 오래된 데이터 기준

정적 fallback 묶음에는 생성 시각 `generatedAt`을 둔다. fallback을 사용한 시점에
`generatedAt`이 7일을 넘었으면 "최근 정보를 불러오지 못해 이전 공개 정보를 보여
드립니다."라는 뜻의 짧은 안내와 기준 날짜를 함께 표시한다.

- 7일 이내 정적 데이터는 fallback 안내만 표시한다.
- 7일을 넘은 정적 데이터는 fallback 안내와 기준 날짜를 표시한다.
- 공연처럼 종료 시각이 있는 항목은 현재 시각에 이미 끝났다면 최신 목록에서 제외한다.
- Columns, 작품 설명처럼 종료 시각이 없는 공개 콘텐츠는 오래됐다는 이유만으로 숨기지
  않는다.
- 화면 문구는 데이터가 최신이라고 보장하지 않으며, 사용자가 다시 시도할 수 있게 한다.

## 구현 전 확인표

- [ ] read adapter가 성공, 정상 빈 결과, 오류, 응답 형식 오류를 구분한다.
- [ ] 공개 쿼리와 정적 파일에서 `draft`와 `private`가 제외된다.
- [ ] 정적 데이터에 `generatedAt`이 있고 7일 경계를 자동 검증한다.
- [ ] 종료된 기간형 콘텐츠를 fallback 목록에서 제외한다.
- [ ] fallback 중에도 기존 상세·다운로드 링크가 동작한다.
- [ ] Supabase와 정적 데이터가 모두 실패하는 화면을 검증한다.
