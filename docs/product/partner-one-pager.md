# in C 현재 상태 한 장 요약

기준일: 2026-07-21

대상: 투자·지원·협력 담당자와 초기 사용자 인터뷰 참여자

## 한눈에 보는 in C

in C는 클래식에 대해 모르는 것을 숨기지 않고, 각자가 들은 것과 듣지 못한 것을
솔직하게 나눌 수 있는 소통 공간을 만드는 서비스다. 사용자는 Columns에서 질문을
찾고, Compositions에서 작품과 악보를 살펴본 뒤, Chromatics에서 악보를 직접 열고
편집할 수 있다.

현재 앱과 정적 사이트는 내부 QA와 사전 출시 후보로 검토할 수 있는 **조건부 가능**
상태다. 이는 정식 출시나 모든 기능의 완성을 뜻하지 않는다. 운영 백엔드와 실제 배포
환경 등은 별도 확인과 승인이 필요하다.

## 현재 확인 가능한 제품 표면

| 표면 | 지금 확인할 수 있는 것 | 현재 범위 |
| --- | --- | --- |
| Columns | 클래식을 듣기 위한 질문과 작품·악보 연결 | 공개된 핵심 콘텐츠 |
| Compositions | 출처를 확인한 단선율 악보, 작품 정보, MusicXML 동선 | 공개된 악보 탐색 표면 |
| Works | 작품을 중심으로 악보와 Columns를 잇는 관계 | Compositions 안의 관계 데이터 |
| Concerts | 작품과 연결할 공연 정보와 감상 안내 | 실제 예약이 아닌 프리뷰 후보 |
| Creators | 작곡가·연주자·기획자 등 관련 인물 정보 | 독립 프로필이 아닌 보조 정보 |
| Classes | 작품과 Columns를 따라가는 학습 안내 | 신청·결제가 없는 정보형 후보 |
| Chromatics | 단일 파트·단성부 중심 악보 편집, MusicXML 열기·저장, PDF 변환 | 데스크탑 사전 출시 후보 |

제품의 중심은 기능을 많이 나열하는 데 있지 않다. 질문, 작품, 악보, 사람과 공연을
연결해 사용자가 자신의 언어로 감상을 시작하도록 돕는 데 있다. 자세한 방향은
[제품 포지셔닝](positioning.md)에서 확인할 수 있다.

## 검증 근거

- 앱은 타입 검사, 자동 테스트, 화면 구성 요소 테스트, 주요 사용자 흐름과 악보 화면
  비교 검증 경로를 갖추고 있다.
- 정적 사이트는 빌드, 콘텐츠 관계, 분석 이벤트, 검색 노출 정보를 확인하는 명령을
  갖추고 있다.
- 2026-07-15 품질 기록에서는 위 핵심 자동 검증이 통과했다. 새 출시 후보에서는 같은
  검증과 사람의 수동 확인을 다시 실행해야 한다.
- 현재 릴리즈 판단과 기능별 근거는 [현재 상태 보고서](../quality/current-state-report.md)와
  [출시 준비 체크리스트](../quality/release-readiness-checklist.md)에서 추적한다.

자동 검증 통과는 설치 파일, 실제 운영 주소, 모든 브라우저가 확인되었다는 뜻이 아니다.
실제 공개 전에는 패키징된 앱과 운영 주소를 사람이 다시 확인해야 한다.

## 남은 위험과 승인 필요 항목

| 항목 | 현재 판단 | 다음 조건 |
| --- | --- | --- |
| 운영 백엔드·계정·서버 데이터 | 보류 | #316 범위의 Supabase 환경, 인증, 접근 정책을 승인 후 검증 |
| 같은 오선의 여러 독립 성부 | 제한 | #93의 완성된 편집 흐름 전까지 단성부를 지원 범위로 안내 |
| 여러 악기 파트와 합주보 | 제한 | #94 완료 전까지 단일 파트를 지원 범위로 안내 |
| Windows 개발 환경 | 보류 | #8의 개발 서버 경고 영향을 실제 Windows에서 확인 |
| 실제 배포와 운영 주소 | 미확인 | 배포 승인 뒤 운영 주소 점검과 되돌리기 절차 확인 |

따라서 지금 요청할 수 있는 협력은 제품 방향 검토, 초기 사용자 인터뷰, 콘텐츠·악보
관계에 대한 피드백, 제한된 사전 출시 QA다. 운영 전환, 개인정보를 다루는 기능,
정식 출시 확정은 이 문서만으로 요청하거나 승인하지 않는다. 상세 위험은
[위험 목록](../quality/risk-register.md)에서 확인한다.

## 이 문서의 역할

이 문서는 현재 제품과 검증 상태를 5분 안에 공유하기 위한 안내서다. #279와 #290의
창업오디션 제출 자료를 대신하지 않으며, 시장 규모·재무 계획·개인 창업자 서사를
포함하지 않는다. 제출 문서에는 별도로 확인한 출처와 최신 수치가 필요하다.
