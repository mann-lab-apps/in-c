# 초기 베타 사용자 모집 범위

작성일: 2026-07-16

## 목적

초기 베타 모집은 계정, 결제, 커뮤니티 기능을 열기 전 공개 사이트와 GitHub
제보만으로 확인할 수 있는 사용 흐름을 검증한다. 목표는 많은 가입자를 모으는
것이 아니라 Columns, Compositions, Chromatics 첫 악보 흐름에서 어디가 막히는지
작게 확인하는 것이다.

## 모집 대상

| 대상 | 이유 | 우선 질문 |
| --- | --- | --- |
| 클래식 입문자 | Columns와 작품 질문이 부담을 낮추는지 확인 | 읽고 나서 무엇을 들어보고 싶어졌는가 |
| 단선율 악보 작성자 | Chromatics의 첫 악보 성공 여부 확인 | 새 악보 생성부터 MusicXML 저장까지 막힌 곳은 어디인가 |
| 음악 교육자 | Classes와 악보 예제가 수업 보조 자료가 되는지 확인 | 수업에서 바로 쓸 수 있는 안내는 무엇인가 |
| 소규모 공연 주최자 | Concerts 프리뷰와 홍보 반복 부담 가설 확인 | 공연 정보를 여러 채널에 다시 쓰는 데 걸리는 시간은 얼마인가 |

## 모집 경로

- 홈 또는 Chromatics 다운로드 근처에 짧은 베타 안내를 둘 수 있다.
- Columns 끝부분에는 글을 읽은 뒤 남길 수 있는 피드백 질문을 우선한다.
- Compositions 상세에는 Chromatics 열기, MusicXML 다운로드, 실패 제보를 연결한다.
- 연락처 수집이 필요하면 공개 사이트 안에 임시 폼을 만들기 전 개인정보 고지와
  저장 위치를 다시 결정한다.

## 수집하는 정보와 수집하지 않는 정보

| 구분 | 허용 | 제외 |
| --- | --- | --- |
| 공개 사이트 analytics | 페이지, 공개 콘텐츠 slug, 다운로드/열기 클릭 | 이름, 이메일, 앱 내부 편집 내용 |
| GitHub issue 제보 | 운영체제, 앱 버전, 재현 순서, 공개 예제 파일 | 개인 연락처, 비공개 악보, 민감 파일 |
| 인터뷰/관찰 테스트 | 익명 코드, 대상군, 요약 인사이트 | 원문 녹취, 실명, 연락처 |

## 노출 위치 후보

| 위치 | 문구 방향 | 준비 조건 |
| --- | --- | --- |
| `site/index.html` 다운로드 영역 | "첫 악보 흐름을 함께 확인해 주세요" | 지원 범위 페이지와 privacy 고지 연결 |
| `site/compositions.html` 상세 | "열어보고 막힌 지점을 알려 주세요" | MusicXML 지원 범위 안내 연결 |
| Columns 상세 하단 | "읽고 나서 떠오른 질문" | 선택지 기반 피드백 유지 |
| GitHub issue template | 버그와 콘텐츠 오류를 구분해 제보 | 공개 이슈 개인정보 경고 포함 |
