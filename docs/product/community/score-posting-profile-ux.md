# 커뮤니티 악보 게시와 공개 프로필 UX

## 악보 게시 흐름

1. 사용자가 Chromatics 또는 웹 편집기에서 `커뮤니티에 게시`를 선택한다.
2. 게시 폼에서 제목, 설명, 태그, 난이도, 공개 범위, 라이선스를 입력한다.
3. MusicXML 원본과 선택 PDF 미리보기를 확인한다.
4. 저작권 체크리스트를 확인하고 게시 또는 검수 요청을 보낸다.
5. 공개 후 상세 페이지에서 열기, 다운로드, 공유, 신고가 가능하다.

## 게시 필드

| 필드 | 필수 | 설명 |
| --- | --- | --- |
| `title` | 예 | 악보 제목 |
| `description` | 아니오 | 짧은 소개 |
| `tags` | 아니오 | 장르, 편성, 난이도 힌트 |
| `difficulty` | 아니오 | `beginner`, `intermediate`, `advanced` |
| `visibility` | 예 | `private`, `unlisted`, `public` |
| `license` | 예 | 원본/편곡/채보 사용 조건 |
| `sourceWorkId` | 아니오 | 작품 페이지 연결 |
| `musicXmlAssetId` | 예 | 편집 가능한 원본 |
| `pdfAssetId` | 아니오 | 미리보기 또는 다운로드 파일 |
| `reviewStatus` | 예 | 공개 전 검수 상태 |

## 공개 프로필

프로필은 사용자가 만든 악보와 연주회 홍보를 한 곳에서 발견하게 하는 최소
허브다.

- 표시 이름, 소개, 지역 또는 소속, 외부 링크
- 공개 악보 목록
- 주최 또는 참여 공연 홍보 목록
- Creator 프로필과 연결할 수 있는 선택 필드
- 신고 또는 관리자 숨김 상태

## 1차 상호작용

- Chromatics에서 열기
- MusicXML 다운로드
- 링크 공유
- 북마크
- 신고

좋아요 수나 댓글은 MVP 범위에서 제외하고, 이벤트 계측만 먼저 남긴다.

## 운영 기능

작성자는 게시물을 비공개로 전환하거나 삭제할 수 있다. 관리자는 신고 누적,
저작권 문제, 스팸 사유로 블라인드 처리할 수 있고, 작성자에게 반려 사유를
표시한다.

## 후속 구현

- `ScorePost` API contract
- 게시 폼과 공개 프로필 화면
- 저작권 체크리스트 validation
- 북마크/공유/신고 이벤트 계측
