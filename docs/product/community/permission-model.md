# 권한 모델과 사용자 역할

## 역할

| 역할 | 설명 |
| --- | --- |
| `guest` | 로그인하지 않은 사용자 |
| `user` | 기본 로그인 사용자 |
| `creator` | 공개 프로필과 작품/악보 연결이 필요한 창작자 |
| `organizer` | 연주회 홍보를 등록하는 사용자 또는 단체 |
| `moderator` | 신고와 검수 큐를 처리하는 운영자 |
| `admin` | 결제, 권한, 운영 설정까지 관리하는 관리자 |

MVP에서는 `user`를 기본으로 두고, `creator`와 `organizer`는 프로필 속성 또는
승인 상태로 확장할 수 있다.

## 접근 정책

- `private`: 소유자와 관리자만 접근
- `unlisted`: 링크를 가진 사용자는 읽기 가능
- `public`: 공개 탐색과 검색 가능
- `team`: MVP 이후 단체 계정이 생기면 추가

## 권한 실패 처리

API는 권한 실패를 `401 unauthenticated`와 `403 forbidden`으로 구분한다.
프론트는 401에서는 로그인 CTA를, 403에서는 `이 작업을 할 권한이 없습니다`와
소유자/관리자 문의 경로를 보여준다.

## 결제와 홍보 권한

결제 완료는 홍보 노출 권한의 충분조건이 아니다. 유료 홍보는 결제 성공과 검수
승인이 모두 있어야 공개 placement에 노출된다. 환불 또는 검수 반려 시 노출
권한은 즉시 중단된다.

## 후속 구현

- API authorization middleware
- 리소스별 ownership check
- 관리자 역할 변경 감사 로그
- 프론트 권한 실패 메시지 컴포넌트
