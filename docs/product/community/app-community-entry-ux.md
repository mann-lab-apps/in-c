# 앱 내 커뮤니티 얕은 연동 UX

## 방향

Electron 앱은 편집 도구로 남고, 커뮤니티는 웹/API 표면으로 연결한다. 앱 안에
완전한 커뮤니티 피드를 넣지 않고, 작업을 방해하지 않는 진입점과 상태 표시만
제공한다.

## 진입점

| 위치 | 노출 내용 | 로그인 필요 |
| --- | --- | --- |
| 시작 화면 | 인기 악보, 새 Columns, 웹 기능 안내 | 아니오 |
| 시스템 탭 | 계정 상태, API 연결 상태 | 일부 |
| 파일 저장 후 | 커뮤니티에 게시 CTA | 예 |
| 사이드 패널 | 현재 악보와 관련 작품/Columns | 아니오 |
| 배너 슬롯 | 공연 홍보 또는 릴리즈 공지 | 아니오 |

## 로그인 전/후

로그인 전에는 탐색과 읽기 중심 링크만 제공한다. 게시, 북마크, 내 게시물,
결제 내역은 로그인 후에만 활성화한다. 로그인 필요 상태에서는 현재 편집 내용을
잃지 않도록 외부 브라우저 인증 후 앱으로 돌아오는 흐름을 우선한다.

## 외부 웹 이동

MVP는 앱 내부 WebView보다 외부 브라우저를 기본으로 한다. 결제, 로그인, 커뮤니티
게시처럼 보안과 웹 세션이 중요한 흐름을 브라우저에서 처리하고, 앱은 deep link
또는 복귀 안내로 상태를 갱신한다.

## 방해 방지 원칙

- 편집 중 자동 팝업을 띄우지 않는다.
- 배너는 닫기와 다시 보지 않기를 제공한다.
- 저장, 내보내기, 재생 등 핵심 편집 버튼보다 커뮤니티 CTA가 우선하지 않는다.
- 오프라인이면 조용히 fallback 문구만 보여준다.

## 후속 구현

- 앱 시작 화면 배너 슬롯
- 계정 상태 표시 컴포넌트
- `커뮤니티에 게시` CTA와 브라우저 이동
- API 실패와 오프라인 fallback 상태
