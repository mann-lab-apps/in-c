# 커뮤니티 게시물 검수와 신고/블라인드 운영 흐름

## 신고 사유

사용자는 공개 악보, 프로필, 공연 홍보에서 다음 사유로 신고할 수 있다.

- 저작권 또는 라이선스 문제
- 허위 공연 정보
- 스팸 또는 반복 홍보
- 혐오/괴롭힘/부적절한 표현
- 개인정보 노출
- 기타 운영자 검토 필요

## 처리 상태

| 상태 | 의미 |
| --- | --- |
| `open` | 신고 접수 |
| `triaged` | 운영자가 사유와 우선순위를 확인 |
| `hidden` | 임시 블라인드 |
| `resolved` | 조치 완료 |
| `dismissed` | 신고 사유 없음 |
| `appealed` | 작성자 이의제기 |

## 자동 블라인드 기준

MVP에서는 완전 자동 삭제를 하지 않는다. 같은 콘텐츠에 신고가 반복되거나
저작권/개인정보 사유가 포함되면 임시 블라인드 후보로 표시하고, 운영자 확인
후 공개 여부를 결정한다.

## 연주회 홍보 검수

홍보 콘텐츠는 공개 전 검수 큐를 거친다. 반려 사유 템플릿은 다음을 포함한다.

- 공연 일시 또는 장소가 불명확함
- 예매/정보 링크가 열리지 않음
- 이미지 사용 권리가 확인되지 않음
- 프로그램 또는 출연자 정보가 과장됨
- 유료 홍보 표시 정책과 맞지 않음

## 관리자 화면 요구사항

- 신고 목록: 상태, 사유, 신고 수, 대상 유형, 최근 신고 시각
- 상세: 원문, 작성자, 관련 파일, 신고 내역, 결제 여부
- 액션: 숨김, 공개 복구, 반려, 작성자 알림, 환불 검토 요청
- 필터: 콘텐츠 유형, 상태, 사유, 유료 홍보 여부
- 감사 로그: 운영자, 액션, 시각, 사유

## 작성자 알림

작성자에게는 조치 결과와 수정 가능한 항목을 짧게 보여준다. 법률적 확정 표현은
피하고, 필요한 경우 `권리 상태를 확인할 수 없어 보류되었습니다`처럼 운영
판단으로 표현한다.

## 후속 구현

- `ModerationCase` API schema
- 신고 버튼과 신고 폼
- 관리자 검수 큐
- 작성자 이의제기와 운영 로그
