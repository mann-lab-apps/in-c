# Codex ATDD Goal Prompts

작성일: 2026-07-19
관련 이슈: #393

Codex 할당량을 충분히 쓰되 사용자의 인지부하를 줄이기 위한 목표모드 프롬프트다.
일단 두 개만 사용하고, 매주 결과를 보며 조금씩 고도화한다.

## 공통 작성 원칙

- 모든 GitHub 이슈 제목, 이슈 본문, 작업 요약, 검증 결과, 후속 문서는 한글로 작성한다.
- 사용자가 5분 안에 이해할 수 있도록 짧고 쉬운 문장을 쓴다.
- 기술 용어는 필요할 때만 쓰고, 처음 나올 때는 사용자가 이해할 수 있는 말로 풀어쓴다.
- 파일 경로, 명령어, API 경로, 코드 식별자는 원문 표기를 유지한다.
- 출력은 후보, 선택 이유, 변경점, 검증 결과, 다음 행동을 눈에 들어오는 단위로 나눈다.

## 1. 이슈 생성 프롬프트

```text
목표: 이 프로젝트에서 ATDD로 처리하기 좋은 새 이슈 후보를 찾아, 사용자가 바로 등록하거나 검토할 수 있는 GitHub 이슈 초안을 작성한다.

범위:
- 현재 프로젝트 상태를 확인한다.
- 서비스 코드, site, workflow, secret, 배포 설정은 수정하지 않는다.
- 새 구현을 하지 않는다.
- 중복 이슈를 만들지 않는다.
- 사용자가 검토할 수 있는 새 이슈 초안만 만든다.
- GitHub 이슈 제목과 본문은 한글로, 비개발자도 이해할 수 있게 작성한다.

진행:
1. 현재 프로젝트를 훑고 새 이슈 후보 3개를 고른다.
   - ATDD 인수 기준과 검증 기준을 만들 수 있다.
   - 사용자의 인지부하를 줄이는 작은 단위다.
   - production, secret, 결제, 배포, 외부 계정 변경이 필요하지 않다.
   - 기존 작업과 중복되지 않는다.
2. 후보 3개 중 가장 좋은 1개를 선택하고 이유를 짧게 설명한다.
3. 선택한 후보를 GitHub 이슈 초안으로 작성한다.
4. 이슈 안에 ATDD 인수 기준과 실패 가능한 검증 후보를 포함한다.
5. 이슈 선정 및 해결 프롬프트가 바로 사용할 목표 문장을 만든다.

출력:
- 후보 3개와 선택 이유
- 선택한 새 이슈 후보
- GitHub 이슈 제목
- GitHub 이슈 본문 초안
- ATDD 인수 기준
- 실패 가능한 검증 기준
- 중복 확인 결과
- 이슈 선정 및 해결 프롬프트에 넘길 목표 문장
- 사용자가 5분 안에 검토할 부분

완료 조건:
- 구현 변경 없이 새 이슈 초안이 남는다.
- 다음 단계에서 바로 사용할 목표 문장이 있다.
- 결정이 필요한 항목은 질문으로 남긴다.
```

## 2. 이슈 선정 및 해결 프롬프트

```text
목표: 이 프로젝트의 열린 GitHub 이슈 중 인지부하가 낮고 치명적이지 않은 후보를 하나 골라 ATDD 방식으로 해결한다.

범위:
- 우선 문서, 사업/운영 기준, 진행상황 문서, 검증 기준, 작은 사용자-facing 문구처럼 위험이 낮은 작업을 고른다.
- 서비스 코드, site, workflow, secret, 배포 설정은 명시 승인 없이 수정하지 않는다.
- 범위가 커지면 구현하지 말고 이슈를 분해한다.
- 작업 요약, 검증 결과, 후속 문서는 한글로 쉽게 작성한다.

진행:
1. 현재 브랜치와 워킹트리를 확인한다.
2. 열린 GitHub 이슈 중 안전한 후보 3개를 고른다.
3. 반드시 가장 적절한 1개만 선택하고 이유를 짧게 설명한다.
   - 나머지 2개는 작업하지 않고 다음 후보로 남긴다.
4. 해당 이슈만 다루는 새 브랜치를 만든다.
5. 먼저 인수 기준을 작성한다.
6. 구현 전에 실패 가능한 검증 기준을 작성한다.
7. 최소 변경만 수행한다.
8. 관련 검증을 실행한다.
9. 변경을 수정하지 말고 독립 리뷰한다.
10. 검증이 통과하면 브랜치를 푸시한다.
11. PR을 만들고, 명시된 기본 머지 전략에 따라 머지한다.
12. 사용자가 5분 안에 볼 확인 경로를 만든다.

출력:
- 후보 3개와 선택 이유
- 선택한 이슈/목표
- 작업하지 않고 남긴 후보 2개
- 인수 기준
- 검증 기준
- 변경 파일
- 검증 결과
- 독립 리뷰 결과
- 브랜치 이름
- 푸시/PR/머지 결과
- 사용자가 5분 안에 확인할 경로
- 다음 후보

완료 조건:
- 작업이 작게 끝나거나, 범위가 크면 분해 이슈/프롬프트로 전환된다.
- 한 번의 실행에서는 이슈 1개만 작업한다.
- 검증 결과가 남는다.
- 각 이슈 작업은 별도 브랜치에서 완료되고, 푸시와 머지 결과가 남는다.
- 사용자가 직접 볼 것은 최대 3개로 제한된다.
```

## 주간 고도화

매주 한 번만 아래 질문으로 두 프롬프트를 조정한다.

```text
이번 주 ATDD 목표모드 결과를 보고 두 프롬프트를 조금만 고도화해줘.

기준:
- 내가 검토하는 데 오래 걸린 부분은 줄인다.
- 자동 선택이 잘 맞았던 조건은 유지한다.
- 범위가 커졌던 조건은 제외 조건으로 추가한다.
- 출력 형식은 더 짧고 명확하게 만든다.

출력:
- 유지할 점
- 줄일 인지부하
- 바꿀 문장
- 다음 주 실험할 작은 변경 1개
```
