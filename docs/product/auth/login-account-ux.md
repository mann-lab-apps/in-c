# 로그인/회원가입 UX와 계정 상태 UI

## 인증 방식

MVP UX는 이메일 매직링크를 기준으로 설계한다. 사용자는 이메일을 입력하고,
메일의 링크를 열어 세션을 만든다. 비밀번호 재설정 화면은 MVP 범위에서 제외한다.

## 기본 흐름

1. 사용자가 로그인 필요 기능을 선택한다.
2. 현재 작업을 저장하거나 임시 보존한다.
3. 이메일 입력 화면을 보여준다.
4. 매직링크 전송 완료와 재전송 버튼을 보여준다.
5. 인증 완료 후 원래 작업으로 돌아온다.

## 계정 상태

| 상태 | UI |
| --- | --- |
| `signedOut` | 로그인 CTA, 로그인 없이 가능한 기능 안내 |
| `linkSent` | 이메일 확인 안내, 재전송 버튼 |
| `signedIn` | 표시 이름, 이메일, 로그아웃 |
| `expired` | 세션 만료 안내, 다시 로그인 |
| `offline` | 오프라인 안내, 로컬 편집 가능 상태 |

## 로그인 없이 가능한 기능

- 로컬 악보 편집
- MusicXML 열기와 저장
- 공개 Columns와 Compositions 탐색
- 공개 악보 보기

로그인이 필요한 기능은 커뮤니티 게시, 북마크, 내 악보 저장, 공유 링크 관리,
결제 내역 확인이다.

## Electron 앱

Electron 앱은 외부 브라우저 로그인을 기본으로 한다. 인증 후 앱으로 돌아오는
deep link가 준비되기 전에는 `로그인 완료 후 앱으로 돌아와 새로고침` 안내를
제공한다. 로그인 실패가 로컬 편집을 막아서는 안 된다.

## 웹 편집기

웹 편집기는 저장, 공유, 공개 범위 변경 시 로그인을 요구한다. 익명 사용자가
편집 중이면 브라우저 세션에 임시 문서를 유지하고, 로그인 후 계정 문서로
이관한다.

## 후속 구현

- 로그인 모달과 매직링크 전송 화면
- 계정 상태 표시 컴포넌트
- 세션 만료 toast와 재로그인 CTA
- Electron 외부 브라우저 인증 복귀 흐름
