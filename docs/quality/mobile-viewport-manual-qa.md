# 모바일 viewport 사이트 수동 QA

공개 사이트의 핵심 흐름이 작은 화면에서도 읽고 조작할 수 있는지 15분 안에 확인하는
수동 검증 기준이다. `viewport`는 브라우저에서 페이지가 보이는 화면 영역을 뜻한다.
실행하지 않은 항목은 통과로 추정하지 않고 `미실행`으로 기록한다.

## 인수 기준

- 360px, 390px, 768px 너비에서 같은 체크리스트를 실행할 수 있다.
- 홈 CTA, 공통 nav, Compositions 필터, Columns·Compositions 상세 패널의 기대 결과가
  짧은 문장으로 정리되어 있다.
- 각 화면 너비의 판정과 스크린샷을 같은 기록 양식에 남길 수 있다.
- 실패 시 재현 URL, 사용자 영향, 후속 이슈를 기록할 수 있다.

## 실패 가능한 검증 기준

- 페이지 전체에 의도하지 않은 가로 스크롤이 생기면 실패다.
- 글자, 버튼, 카드 또는 상세 패널이 겹치거나 잘려 핵심 내용을 읽을 수 없으면 실패다.
- nav, CTA, 필터 또는 상세 열기·닫기를 터치로 실행할 수 없으면 실패다.
- 필터 결과나 query 상세 상태가 화면 크기를 바꾼 뒤 사라지면 실패다.
- 화면 너비, 확인 URL, 판정, 스크린샷 중 하나라도 근거에 없으면 통과로 기록하지 않는다.

## 실행 전 준비

1. `npm run site:build`를 실행한다.
2. `npm run site:preview`로 로컬 사이트를 연다.
3. 브라우저 개발자 도구의 반응형 화면에서 높이는 800px 이상, 확대는 100%로 둔다.
4. 360px, 390px, 768px 너비를 차례로 선택한다.
5. 각 너비에서 아래 체크리스트를 처음부터 실행한다.

프로덕션 확인은 배포 승인을 받은 뒤에만 진행한다. 실제 모바일 기기를 사용했다면
CSS 화면 너비와 기기·브라우저 이름을 함께 적는다.

## 공통 체크리스트

| 확인 항목 | 실행 방법 | 기대 결과 | 필수 스크린샷 |
| --- | --- | --- | --- |
| 홈 CTA | 홈 첫 화면에서 `Columns`, `Compositions`, 앱 다운로드로 이어지는 링크를 확인하고 하나씩 연다. | 핵심 문구와 CTA가 겹치지 않고, 링크를 터치해 해당 위치나 페이지로 이동할 수 있다. | 홈 첫 화면 |
| 공통 nav | 홈, Columns, Compositions, Community에서 공통 nav를 확인하고 각 링크를 연다. | nav 항목이 잘리지 않고, 현재 페이지와 이동할 페이지를 구분할 수 있다. | nav가 가장 좁게 보이는 화면 |
| Compositions 필터 | `compositions.html`에서 검색어, 난이도, 태그 필터를 각각 바꾸고 초기화한다. | 필터와 결과 수를 읽을 수 있고, 입력·선택·초기화가 가능하며 카드가 화면 밖으로 넘치지 않는다. | 필터 적용 결과 |
| Compositions 상세 | 악보 카드를 열고 상세 내용을 읽은 뒤 닫는다. `?score=<slug>` URL도 새로 연다. | 상세 패널의 제목, 설명, 주요 링크와 닫기 동작을 사용할 수 있고 query로 연 상세가 유지된다. | 상세 패널 전체 구조 |
| Columns 상세 | `columns.html`에서 항목을 열고 `?column=<slug>` URL을 새로 연다. | 본문을 가로 스크롤 없이 읽을 수 있고 query로 연 상세가 유지된다. | 제목과 본문 시작 부분 |

스크린샷에는 브라우저 UI 또는 메모로 화면 너비를 식별할 수 있어야 한다. 개인정보,
접근 토큰, 로컬 파일 경로는 포함하지 않는다. 같은 문제가 여러 너비에서 반복되면 대표
스크린샷 하나와 영향받는 너비 목록으로 합칠 수 있다.

## 판정 기준

- `통과`: 해당 너비의 모든 항목이 기대 결과와 같고 필수 근거가 있다.
- `조건부 통과`: 핵심 흐름은 가능하고 작은 표시 차이만 있으며 후속 이슈가 있다.
- `실패`: 실패 가능한 검증 기준 중 하나에 해당하거나 핵심 흐름이 막힌다.
- `미실행`: 확인하지 않았거나 필수 근거가 없다.

한 너비라도 `실패`이면 전체 판정을 `통과`로 기록하지 않는다.
