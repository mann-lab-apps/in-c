# 사이트 접근성 수동 QA

공개 사이트의 핵심 화면을 키보드와 화면 읽기 도구로 사용할 수 있는지 15분 안에
확인하는 수동 검증 기준이다. 화면 읽기 도구는 화면의 글과 구조를 음성으로 읽어 주는
기능이다. 실행하지 않은 항목은 통과로 추정하지 않고 `미실행`으로 기록한다.

## 인수 기준

- 홈, Columns, Compositions, 제품 페이지, 개인정보 페이지에 같은 기준을 적용할 수 있다.
- 키보드 이동, 현재 초점, 제목 구조, 링크·버튼·입력 이름을 짧은 순서로 확인할 수 있다.
- 페이지별 판정과 확인 근거를 같은 기록 양식에 남길 수 있다.
- 실패 시 재현 URL, 사용자 영향, 후속 이슈를 기록할 수 있다.

## 실패 가능한 검증 기준

- `Tab`과 `Shift+Tab`만으로 주요 링크, 버튼, 입력 항목에 갈 수 없으면 실패다.
- 현재 초점이 보이지 않거나 화면에 보이는 순서와 크게 다르게 이동하면 실패다.
- 화면 읽기 도구에서 페이지 제목과 주 제목을 알 수 없거나 제목 단계가 내용을
  이해하기 어렵게 건너뛰면 실패다.
- 아이콘 링크, 버튼, 검색·필터 입력의 이름이나 상태를 알 수 없으면 실패다.
- 키보드로 연 상세 내용을 닫거나 이전 위치로 돌아갈 수 없으면 실패다.
- 확인 URL, 브라우저, 화면 읽기 도구, 판정 중 하나라도 근거에 없으면 통과로 기록하지 않는다.
- 이 문서의 상대 링크가 없거나 `git diff --check`가 실패하면 완료로 보지 않는다.

## 실행 전 준비

1. `npm run site:build`를 실행한다.
2. `npm run site:preview`로 로컬 사이트를 연다.
3. 마우스를 사용하지 않고 `Tab`, `Shift+Tab`, `Enter`, `Space`, `Esc`를 사용한다.
4. macOS는 VoiceOver, Windows는 Narrator, Linux는 Orca 중 사용할 수 있는 화면 읽기
   도구 하나를 켠다.
5. 아래 페이지를 차례로 확인한다. 제품 페이지는 `community.html`에서 목록과 상세를
   각각 확인한다.

프로덕션 확인은 배포 승인을 받은 뒤에만 진행한다. 자동 검사 결과만으로 수동 확인을
통과 처리하지 않는다.

## 공통 체크리스트

모든 페이지에서 먼저 아래 항목을 확인한다.

| 확인 항목 | 실행 방법 | 기대 결과 |
| --- | --- | --- |
| 페이지와 주 제목 | 페이지를 연 뒤 화면 읽기 도구의 제목 탐색을 실행한다. | 페이지 용도와 주 제목을 알 수 있고 제목 순서로 내용을 이해할 수 있다. |
| 키보드 이동 | 처음부터 `Tab`으로 끝까지 이동하고 `Shift+Tab`으로 되돌아간다. | 주요 행동을 빠뜨리지 않고 화면의 읽기 순서에 맞게 이동한다. |
| 현재 초점 | 이동할 때마다 현재 위치를 눈으로 확인한다. | 초점 테두리나 같은 수준의 표시가 배경과 구분된다. |
| 이름과 상태 | 각 링크, 버튼, 입력에서 화면 읽기 도구가 읽는 이름과 상태를 듣는다. | 보이는 목적과 이름이 맞고 현재 페이지, 선택, 펼침 상태를 알 수 있다. |
| 확대 | 브라우저를 200%로 확대하고 주요 내용을 확인한다. | 글과 조작 항목이 겹치거나 잘리지 않고 가로 스크롤 없이 핵심 흐름을 사용할 수 있다. |

## 페이지별 체크리스트

| 페이지 | 확인 URL | 추가 실행 방법 | 기대 결과 |
| --- | --- | --- | --- |
| 홈 | `index.html` | 주요 링크를 차례로 열고 앱 다운로드 영역까지 이동한다. | 현재 페이지와 이동 목적을 구분하고 핵심 링크를 키보드로 실행할 수 있다. |
| Columns | `columns.html` | 지도 보기 조절, 공개 칼럼, 준비 중 칼럼을 차례로 확인한다. | 조절 버튼의 이름과 공개 여부를 알 수 있고 공개 칼럼만 열 수 있다. |
| Compositions | `compositions.html` | 검색, 난이도, 태그로 목록을 좁힌 뒤 상세와 다운로드 링크를 확인한다. | 입력 이름, 필터 결과, 선택한 상세, 파일 종류를 화면 읽기 도구로 구분할 수 있다. |
| 제품 페이지 | `community.html` | 목록의 `상세 보기`를 키보드로 열고 브라우저 뒤로 가기를 실행한다. | 선택한 항목의 상세가 읽기 순서에 나타나고 이전 위치로 돌아갈 수 있다. |
| 개인정보 | `privacy.html` | 개인정보, 문의, 운영 안내의 제목을 차례로 탐색하고 링크를 연다. | 각 안내의 경계를 제목으로 구분하고 문의·외부 링크의 목적을 알 수 있다. |

## 판정 기준

- `통과`: 모든 공통·페이지별 항목이 기대 결과와 같고 근거가 있다.
- `조건부 통과`: 핵심 흐름은 가능하고 작은 문제가 있으며 후속 이슈가 있다.
- `실패`: 실패 가능한 검증 기준 중 하나에 해당하거나 핵심 흐름이 막힌다.
- `미실행`: 확인하지 않았거나 필수 근거가 없다.

한 페이지라도 `실패`이면 전체 판정을 `통과`로 기록하지 않는다.
