# 릴리즈 문서

릴리즈 점검 문서는 템플릿과 실제 점검 기록을 구분한다.

## 파일 구조

- `checklist-template.md`: 매 릴리즈 전에 복사해서 사용하는 체크리스트 원본.
  제품, 배포, 테스트 방식이 바뀌면 이 파일을 업데이트한다.
- `checklists/`: 완료된 체크리스트를 저장해야 할 때 사용하는 선택적 보관 위치.
  모든 prerelease 체크리스트를 반드시 저장하지는 않는다.

## 운영 방식

기본적으로 실제 릴리즈 체크리스트는 릴리즈 PR, 이슈, 또는 GitHub Release 준비
메모에 붙여 넣고 완료 상태를 남긴다. 저장소 안에 매번 완료본을 커밋하면 routine
alpha 릴리즈 기록이 문서를 빠르게 복잡하게 만들 수 있기 때문이다.

다음 경우에는 완료본을 `checklists/`에 커밋한다.

- 수동 QA 범위가 평소보다 컸다.
- 배포 중 문제가 있었고 재현 가능한 기록이 필요하다.
- beta, stable, 또는 외부 사용자 영향이 큰 릴리즈다.
- 릴리즈 승인 기록을 저장소에 남겨야 한다.

완료본을 저장할 때는 `checklists/v0.1.0-alpha.4.md`처럼 태그 이름을 파일명으로
쓴다.
