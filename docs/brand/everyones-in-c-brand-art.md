# Everyone's In C Brand Art Experiment

작성일: 2026-07-14

Everyone's In C는 in C의 핵심 제품군이 아니라, 2026년 8월 이후에 검토할 수 있는
브랜드 아트 실험 후보다. 현재 7월 우선순위는 Chromatics 편집기, Compositions,
Works, Columns, Concerts, Creators, Classes의 MVP 연결성을 높이는 것이다.

## 실험 의도

사용자가 짧은 패턴을 추가하고 변주하면서 하나의 living score가 자라나는 경험을
브랜드 아트로 보여준다. 제품 메시지는 "클래식은 완성된 박물관이 아니라, 함께
들어가고 만져볼 수 있는 악보의 세계"에 가깝다.

## Terry Riley `In C` 참조 경계

`In C`는 이름과 아이디어 차원에서 중요한 레퍼런스지만, 이 실험은 Terry Riley의
작품을 복제하거나 웹에서 재현하는 프로젝트가 아니다.

- 사용하지 않는다: 원곡의 53개 패턴, 원곡 악보, 원곡 음원, 특정 연주 녹음,
  현대 출판본의 레이아웃 또는 텍스트.
- 사용할 수 있다: "짧은 패턴이 누적되며 공동의 음악 표면을 만든다"는 추상적
  아이디어, 참여형 악보라는 브랜드 은유.
- 이름은 내부 실험명으로만 쓴다. 공개 제목은 저작권/상표/혼동 가능성을 검토한
  뒤 별도로 정한다.
- 사용자가 만든 패턴은 독립적인 라이선스와 운영 약관을 갖는다.

## 참여 방식 후보

1. Pattern wall
   - 사용자는 1마디 또는 2마디짜리 단선율 패턴을 추가한다.
   - 웹 화면은 패턴 카드를 시간순 또는 조성/리듬별로 배열한다.
   - 재생은 나중 범위로 미루고, 먼저 시각적 living score로 시작한다.

2. Call and response
   - 운영자가 짧은 seed pattern을 제시한다.
   - 사용자는 같은 리듬, 반대 방향 선율, 쉼표 추가 같은 제한 안에서 응답한다.
   - 교육 콘텐츠와 Classes로 연결하기 쉽다.

3. Monthly community score
   - 매월 하나의 주제를 정해 작은 패턴을 모은다.
   - 선정된 패턴은 Columns 또는 Works의 예시로 소개한다.
   - 실시간 합주가 아니라 비동기 큐레이션으로 운영한다.

## 7월 제품군과의 연결

- Compositions: 사용자가 패턴을 MusicXML로 내려받아 Chromatics에서 열어본다.
- Columns: "패턴 하나를 듣는 법", "반복과 변주의 차이" 같은 짧은 글로 연결한다.
- Classes: 입문 클래스의 실습 과제로 연결할 수 있다.
- Creators: 운영자 또는 참여자의 공개 프로필 범위를 별도 permission model과 맞춘다.

## 제외 범위

- 2026년 7월 구현.
- 실시간 합주 엔진.
- 음원 배포.
- Terry Riley `In C`의 패턴, 악보, 음원 재사용.
- 사용자 저작물 권리 정책 없이 공개 저장소를 운영하는 것.

## 실험 조건

- 최소 20개의 내부 샘플 패턴을 직접 작성한다.
- 모든 샘플은 Chromatics에서 열 수 있는 단선율 MusicXML로 저장한다.
- 사용자 제출을 받기 전, 저작권/라이선스 안내와 삭제 요청 흐름을 만든다.
- 공개 전 title, description, reference note가 Terry Riley 작품과 혼동되지 않는지
  다시 검토한다.
